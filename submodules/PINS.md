# Submodule pins

This file is the authoritative record of which commit each bundled submodule is
expected to be at.  The CMake configure step reads it and fails immediately if
any submodule HEAD does not match.

Each entry contains:
- **commit** — the exact 40-character git SHA-1 expected at submodule HEAD.
- **tag** — the nearest upstream release tag (informational).
- **pinned** — the date this entry was last updated (YYYY-MM-DD).
- **tree-sha256** — SHA-256 of the submodule's directory tree (extra integrity
  layer beyond git's own SHA-1, which has known weaknesses).
- **rationale** — why this version was chosen.

To upgrade a submodule, run:

```
scripts/update_submodule.sh <name> <new-commit>
```

That script updates the submodule checkout, recomputes the tree hash, and
rewrites the entry below.  Do not edit this file by hand.
